## 2. 會議頁（主頁）

### 2.1 初始狀態（無進行中 Session）

```
┌─────────────────────────────────────┐
│                                     │
│     🎙️ [開始錄音]    📁 [匯入音檔]   │
│                                     │
│─────────────────────────────────────│
│  歷史紀錄                            │
│                                     │
│  📋 2026-04-04 14:30 — 週會（已匯出） │
│  📋 2026-04-03 10:00 — 站會（待審閱） │
│  📋 2026-04-02 15:00 — 客戶會議      │

- 歷史紀錄列表 → 點擊進入該 Session 的審閱頁（Session 格式見 [[data_schema#5. Session（單次會議）]]）

### 2.4 工作區

Pipeline 處理完成後進入工作區。工作區是這個 App 的核心介面，AI 產出後使用者可即時編輯修正。

#### 響應式佈局

使用情境：半邊螢幕多工、外接螢幕全寬、或拉到副螢幕獨立使用。佈局依視窗寬度自動切換：

**寬視窗（≥ 1400px）— 三欄並排**

```
┌─────────────────────────────────────────────────────┐
│  會議 — 2026-04-04 14:30     時長：01:02:15         │
├─────────────┬──────────────────┬────────────────────┤
│  📜 逐字稿   │  💡 會議重點      │  ✅ Action Items   │
│             │                  │                    │
│  帶時間戳    │  AI 產生的       │  結構化待辦清單     │
│  可標記校正   │  重點摘要 + 決議  │  含負責人/期限     │
│  回饋        │  可直接編輯      │  可編輯/新增/刪除   │
│             │                  │                    │
├─────────────┴──────────────────┴────────────────────┤
│  [匯出 Markdown]    [提交回饋]    [刪除]              │
└─────────────────────────────────────────────────────┘
```

**中等視窗（960px ~ 1399px）— 兩欄：逐字稿 + 右側分頁**

```
┌───────────────────────────────────────────┐
│  會議 — 2026-04-04 14:30                  │
├──────────────┬────────────────────────────┤
│  📜 逐字稿    │  [💡 重點] [✅ Actions]     │
│              │                            │
│  帶時間戳     │  ← 分頁切換                │
│  可標記校正    │  當前分頁內容               │
│  回饋         │                            │
│              │                            │
├──────────────┴────────────────────────────┤
│  [匯出 Markdown]    [提交回饋]    [刪除]    │
└───────────────────────────────────────────┘
```

**窄視窗（< 960px）— 單欄分頁**

```
┌─────────────────────────────┐
│  會議 — 2026-04-04 14:30    │
├─────────────────────────────┤
│  [📜 逐字稿] [💡 重點] [✅ Actions] │
│                             │
│  ← 分頁切換                  │
│  當前分頁內容                 │
│                             │
├─────────────────────────────┤
│  [匯出]  [回饋]  [刪除]      │
└─────────────────────────────┘
```

> 欄寬可拖曳調整。使用者調整後的寬度偏好會記住。

#### 三區塊內容

| 區塊 | 內容 | 可編輯 | 資料來源 |
|------|------|--------|----------|
| **逐字稿** | 帶時間戳的逐字稿，校正處高亮標記 | 校正回饋（正確/錯誤/遺漏），不開放全文編輯以保持資料完整性 | [[data_schema#3. 校正結果]] |
| **會議重點** | AI 產生的重點摘要 + 決議事項 | 全文可編輯，AI 產出 vs 使用者修改以不同樣式區分 | [[data_schema#4. 摘要結果]] |
| **Action Items** | 結構化待辦：內容、負責人、期限、優先級 | 表單式編輯各欄位，可新增/刪除/勾選完成 | [[data_schema#4. 摘要結果]] |

#### 逐字稿校正互動

```
[00:05:12] 我們決定用 [Gemma 4]⚡ 來做本地推理
                       ↑ 校正：原始「寶石四」→「Gemma 4」
                         [✅ 正確] [❌ 錯誤] [+ 新增詞條]
```

- 校正處以琥珀色高亮，hover/點擊顯示校正詳情
- 每個校正可直接標記回饋（正確/錯誤）
- 可回報「遺漏」：選取文字 → 右鍵 → 「回報遺漏校正」
- 遺漏回報可「一鍵新增詞條」，降低回饋門檻（參考 [[competitive_analysis_20260404#7.4 RAG 知識庫 — 獨特競爭優勢]]）

#### 會議重點編輯

- AI 原始產出以預設樣式顯示
- 使用者編輯/新增的內容以不同色調區分（參考 Granola 的人機協作概念，見 [[competitive_analysis_20260404#2.6 Granola]]）
- 決議事項獨立區塊呈現，不混在摘要文字中

#### 底部操作

- [匯出 Markdown] → 匯出時包含三個區塊完整內容（格式見 [[data_schema#7. 匯出格式（Markdown）]]）→ 匯出後提示刪除音檔
- [提交回饋] → 批次儲存校正回饋（格式見 [[data_schema#6. 回饋紀錄（Feedback）]]）
- [刪除] → 刪除整個 Session（需二次確認）

## 3. 詞條管理頁

```
┌─────────────────────────────────────┐
│  知識詞條管理          [+ 新增] [匯入] │
│─────────────────────────────────────│
│  🔍 搜尋詞條...        篩選：[全部 ▼] │
│─────────────────────────────────────│
│                                     │
│  詞條        別名          命中/成功  │
│  ──────────────────────────────────│
│  Gemma 4    寶石四,gemma4   15/12   │
│  Claude     克勞德,CLAUDE    8/7    │
│  RAG        拉格,rag        3/1    │
│  ChromaDB   chroma          0/0    │
│                                     │
│─────────────────────────────────────│
│  共 42 筆詞條 │ 平均成功率 78%        │
└─────────────────────────────────────┘
```

**功能：**
- 搜尋：即時過濾詞條
- 篩選：依分類、依效能（高命中/低命中/零命中）
- 新增：開啟詞條編輯表單
- 匯入：批次匯入 YAML 檔案（詞條格式見 [[data_schema#1. 知識詞條（Term）]]）
- 點擊詞條列 → 展開/開啟詳情編輯
- 命中/成功欄位：顯示 `correction_count / success_count`（統計欄位定義見 [[data_schema#1. 知識詞條（Term）]]）

│        成功 10 次 │ 失敗 2 次         │
│                                     │
│           [儲存]  [取消]  [刪除]      │
└─────────────────────────────────────┘
```

```

│  回饋統計                            │
│─────────────────────────────────────│
│                                     │
│  整體校正成功率     78%  ████████░░  │
│  總回饋數           156              │
│  總 Session 數      23              │
│                                     │
│─────────────────────────────────────│
│  ⚠️ 需要關注                         │
│                                     │

│  · ChromaDB（建立 30 天，從未命中）   │

│                                     │
│  高頻遺漏回報（應新增詞條）：          │
│  · 「歐拉瑪」→ Ollama（遺漏 5 次）   │
│  · 「飛特」→ Flet（遺漏 3 次）       │
│                                     │
│─────────────────────────────────────│
│  最近 Session 回饋                   │
│  2026-04-04 — 校正 12/15 正確 (80%) │
│  2026-04-03 — 校正 8/10 正確 (80%)  │
│                                     │
└─────────────────────────────────────┘
```

**功能：**
- 整體統計：成功率、回饋數、Session 數
- 需要關注區：自動分析出需要處理的詞條
  - 低成功率 → 建議補充別名或調整
  - 零命中 → 建議移除（避免向量空間汙染）
  - 高頻遺漏 → 建議新增詞條
- 最近 Session 回饋列表

## 7. 對話框與通知

| 場景 | 類型 | 內容 |
|------|------|------|
| 匯出成功 | 對話框 | 「匯出完成。是否刪除原始音檔？」[刪除] [保留] |
| 刪除 Session | 對話框 | 「確定刪除此會議紀錄？此操作無法復原。」[確定] [取消] |
| Ollama 未啟動 | 通知 | 「無法連線 Ollama，請確認已安裝並啟動。」 |
| 錄音權限 | 通知 | 「需要麥克風權限才能錄音。」 |
| Pipeline 錯誤 | 通知 | 「處理失敗：{錯誤訊息}。請查看日誌。」 |

## 8. 視覺風格

| 項目 | 規格 |
|------|------|
| 色彩主題 | 深色模式為主（護眼），支援淺色切換 |
| 主色調 | 藍灰色系（專業、中性） |
| 強調色 | 校正高亮用琥珀色，成功用綠色，錯誤用紅色 |
| 字型 | 系統預設（Windows: Microsoft JhengHei UI） |
| 最小視窗 | 800 x 600 |
| 響應式 | 三段式：≥1400px 三欄、960~1399px 兩欄、<960px 單欄分頁 |
| 欄寬 | 可拖曳調整，偏好值記憶 |
